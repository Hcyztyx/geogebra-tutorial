# GeoGebra 错误弹窗修复说明

## 问题描述
每个课程点开时都会出现 GeoGebra 错误弹窗，严重影响用户体验。

## 根本原因
GeoGebra API 在调用时如果遇到错误（如对象不存在、方法调用失败等），会：
1. 弹出错误对话框
2. 调用 window.alert 显示错误信息
3. 抛出未捕获的异常

## 修复方案

### 三层防护机制

#### 第一层：安全 API 包装器 (geogebra-safe-api.js)
- 包装所有 GeoGebra API 方法
- 每个方法都有 try-catch 保护
- 错误时返回 null 而非抛出异常
- 记录错误到控制台而非弹窗

#### 第二层：全局错误拦截
- `window.onerror`: 捕获所有脚本错误
- `window.onunhandledrejection`: 捕获 Promise 错误
- 过滤 GeoGebra 相关错误，静默处理

#### 第三层：alert 覆盖
- 覆盖原始 `window.alert` 函数
- 拦截包含 "Error" 或 "错误" 的消息
- 只记录到控制台，不显示弹窗

### 代码修改

#### 1. 新增文件：js/geogebra-safe-api.js
```javascript
window.safeGGB = {
  applet: null,
  setApplet: function(api) { this.applet = api; },
  call: function(method, ...args) {
    try {
      return this.applet[method](...args);
    } catch (error) {
      console.warn(`GeoGebra ${method} error:`, error.message);
      return null;
    }
  },
  // ... 50+ 个安全方法
}
```

#### 2. index.html
```html
<!-- 在 app.js 之前加载安全 API -->
<script src="js/geogebra-safe-api.js"></script>
<script src="js/app.js"></script>
```

#### 3. js/app.js
所有验证方法改用安全 API：
```javascript
// 之前
const x = this.applet.getXcoord(pointName);

// 现在
const safeGGB = window.safeGGB || this;
const x = safeGGB.getXcoord(pointName);
if (x === null) return false;
```
